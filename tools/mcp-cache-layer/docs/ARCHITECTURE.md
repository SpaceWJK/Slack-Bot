# MCP Cache Layer — 시스템 아키텍처

> 최종 수정: 2026-03-10

## 1. 문제 정의

### 1-1. 현상
Wiki/Jira/GDI MCP를 통한 문서 조회/검색/분석 시:
- **시간 과다 소모**: 매 요청마다 MCP 프록시 서버에 HTTP 호출
- **토큰 과다 소모**: 전체 페이지 본문이 LLM 컨텍스트에 삽입
- **중복 비용**: 변경 없는 동일 페이지를 매번 새로 조회

### 1-2. 근본 원인 (3계층)

| 계층 | 원인 | 비용 기여도 |
|------|------|------------|
| **(A) 후보군 확장** | 상위 폴더 기반 재귀 탐색으로 문서 목록 수집 자체가 비쌈 | 중 |
| **(B) 변경 무시 전량 재처리** | 수정 여부 확인 없이 매번 전체 본문 조회 | 대 |
| **(C) 검색/분석 미분리** | "찾아줘"(검색)인데 "전량 요약/분석"을 태움 | 대 |

### 1-3. 영향 범위
- **Claude Code 세션**: MCP 호출 → 컨텍스트 윈도우 토큰 소모 → 조기 compaction
- **Slack Bot /wiki, /gdi**: MCP 호출 → Claude API 토큰 소모 → 응답 지연

## 3. 목표 아키텍처 (TO-BE)

### 3-1. 설계 원칙

1. **기존 프로세스 내 동작**: 새 서비스/포트/데몬 없음 (사내 보안 준수)
2. **기존 코드 최소 변경**: wiki_client.py 등에 캐시 레이어만 삽입
3. **Python 표준 라이브러리 우선**: sqlite3 내장, 외부 의존성 최소
4. **두 경로 공유**: Slack Bot과 Claude Code가 같은 캐시 사용
5. **점진적 확장**: Wiki 먼저 → GDI → Jira 순서로 적용

### 3-3. 소비자별 연동 방식

#### Slack Bot (주 소비자)

```python
# wiki_client.py 변경 (기존 코드에 캐시 레이어 삽입)
from cache_manager import CacheManager

cache = CacheManager("D:/Vibe Dev/QA Ops/mcp-cache-layer/cache/mcp_cache.db")

def get_page_by_title(title, space_key=None, fetch_full=False):
    # 1. 캐시 확인
    cached = cache.get_page(title, space_key)
    if cached and not cached.is_stale():
        return cached.to_page_result(fetch_full)

    # 2. 캐시 miss/stale → 기존 MCP 호출
    result = _mcp_cql_search(title, space_key)

    # 3. 결과 캐시 저장
    cache.put_page(result)
    return result
```

## 4. 캐시 무효화 전략

### 4-1. 복합 검증 (수정일만 쓰면 안 되는 이유 반영)

| 검증 수단 | 감지 가능 | 감지 불가 |
|-----------|----------|----------|
| last_modified | 본문 수정 | 댓글, 첨부파일, 권한 변경 |
| version 번호 | 본문 수정 | 일부 플랫폼에서 메타 변경 |
| content_hash | 본문 변경 (확실) | 첫 조회 시에만 비교 가능 |

**적용 정책:**
- **Primary**: `last_modified` + `version` (메타데이터만으로 빠른 판단)
- **Secondary**: `content_hash` (본문 조회 시 추가 검증)
- **Fallback**: TTL 기반 만료 (기본 24시간, 설정 가능)

### 4-2. 상위/하위 관계 처리

현업자 A/B 공통 지적: "하위 문서 변경이 상위 폴더 수정일에 반영 안 될 수 있음"

**대응:**
- 트리 노드별로 독립적으로 수정일 관리
- 상위 폴더 조회 시 → 하위 노드들의 수정일도 개별 체크
- Delta Sync에서 CQL `lastModified > '{last_sync}'` 로 공간 전체 변경분 확인

## 5. 쿼리 라우터 분류 기준

| 질문 유형 | 키워드 예시 | 데이터 경로 | 비용 |

| **최신성 민감** | "최근 변경", "업데이트된", "오늘" | Delta Sync 실행 후 L2 | 중 |

## 8. 제약 사항 및 리스크

| 제약 | 대응 |
|------|------|
